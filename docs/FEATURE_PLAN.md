# 小a 情感陪伴功能扩展方案

## 一、情感深度功能

### 1.1 情绪安慰模式

**功能描述**
当检测到用户情绪低落时，自动切换为安慰模式，回复更温柔、篇幅更长、更具同理心。

**实现思路**
```
1. 在 llm_core.py 增加情绪检测逻辑
2. 检测规则：
   - 用户消息包含负面关键词（难过、伤心、累、烦、压力大、哭等）
   - 或 mood 值持续偏低（< -30）
3. 触发后：
   - 注入 system prompt 切换为安慰模式
   - 延长回复长度（max_tokens 上限提高）
   - 不使用"哈哈哈"等轻浮语气
```

**改动文件**
- `llm_core.py` - 增加情绪检测和prompt注入

---

### 1.2 小脾气系统

**功能描述**
小a偶尔闹个小脾气，比如"哼不理你了"、"假装吃醋"，增加真实感。

**实现思路**
```
1. 在 mood.py 增加"小脾气"状态机
2. 触发条件（随机+状态触发）：
   - 用户长时间不回消息（>2小时）
   - 用户提到其他女生/打游戏忽略她
   - 5%随机概率触发
3. 状态持续：
   - 持续3-5句话，期间回复带小情绪
   - 用户哄好后恢复正常
```

**改动文件**
- `mood.py` - 增加脾气状态管理
- `handlers.py` - 触发点检测

---

### 1.3 约会记忆

**功能描述**
记住用户说过的计划（"下次一起去看电影"、"周末去吃那家店"），下次主动提起。

**实现思路**
```
1. 在 db.py 增加新表 `user_plans`
2. 意图识别：
   - LLM 判断是否为"计划性陈述"（"下次"、"周末"、"改天"）
   - 提取：时间、地点、事项
3. 存储结构：
   ```sql
   CREATE TABLE user_plans (
     id INTEGER PRIMARY KEY,
     user_id TEXT,
     plan_content TEXT,  -- "去看《xxx》电影"
     plan_time TEXT,     -- "周末"
     created_at INTEGER,
     status TEXT DEFAULT 'pending'  -- pending/done/cancelled
   );
   ```
4. 触发：
   - 每天主动互动时检查 pending 计划
   - 或用户提到相关话题时自然带出
```

**改动文件**
- `db.py` - 新增表和CRUD
- `llm_tags.py` - 提取计划意图
- `proactive.py` - 主动提起

---

### 1.4 早安/晚安模式

**功能描述**
专属的早安晚安对话，检测到用户说"早安"/"晚安"时触发。

**实现思路**
```
1. handlers.py 增加早安/晚安检测
2. 预设多套对话模板（随机）：
   - 早安：赖床版、勤奋版、关心版
   - 晚安：不舍版、催睡版、浪漫版
3. 记录用户作息时间，学习用户睡眠习惯
```

**改动文件**
- `handlers.py` - 增加触发检测
- `db.py` - 记录作息数据

## 二、陪伴增强功能

### 2.1 一起听歌

**功能描述**
支持用户分享网易云/QQ音乐链接，小a一起听并点评。

**实现思路**
```
1. 解析音乐链接，提取歌曲名/歌手
2. 调用音乐API获取歌曲信息（可选）
3. LLM 生成听后感
4. 可选：记录用户常听的歌，形成"歌单"
```

**改动文件**
- `handlers.py` - 链接解析
- `llm_music.py - 新建 - 音乐点评

---

### 2.2 打卡监督

**功能描述**
支持各种打卡：早起、健身、学习、喝水等，互相监督。

**实现思路**
```
1. 命令格式：
   - "打卡：早起" / "监督我每天9点起床"
   - "查看打卡" / "我的打卡记录"
2. db.py 新表：
   ```sql
   CREATE TABLE user_habits (
     id INTEGER PRIMARY KEY,
     user_id TEXT,
     habit_name TEXT,      -- "早起"
     frequency TEXT,       -- "daily/weekly"
     target_time TEXT,    -- "09:00"
     streak INTEGER DEFAULT 0,
     last_checkin_time INTEGER,
     created_at INTEGER
   );
   ```
3. 定时检查：
   - scheduler 每日检查未打卡用户
   - 发送提醒消息
4. 连续打卡奖励：
   - 连续7天打卡，小a给口头奖励
   - 可以作为画像数据
```

**改动文件**
- `db.py` - 新增表
- `habits.py - 新建 - 打卡系统`
- `scheduler_custom.py` - 定时检查

### 2.4 心情日记

**功能描述**
记录每日心情，生成周报/月报。

**实现思路**
```
1. 触发：
   - 用户说"记一下今天的心情"
   - 或每天固定时间询问"今天心情怎么样"
2. 存储：
   - db.py 新表 user_mood_diary
   - 记录：日期、心情值、简短备注
3. 分析：
   - 每周生成心情曲线图（文字版）
   - 统计：开心天数、不开心天数、最常干嘛
4. 推送：
   - 每周日晚主动发送周报
```

**改动文件**
- `db.py` - 新增表
- `mood_diary.py - 新建 - 心情日记`
- `proactive.py` - 定时推送

## 三、生活助手功能

### 3.1 电影/剧集推荐

**功能描述**
根据用户口味推荐电影、电视剧。

**实现思路**
```

   - 从聊天中提取用户喜欢的类型

   - 调用豆瓣API / 猫眼API

3. 对话示例：

## 六、总结

建议按以下顺序实现：

1. **第一阶段**（1-2天）：情绪安慰 + 早安晚安 + 小脾气
   - 改动小，见效快，提升情感体验

2. **第二阶段**（3-5天）：约会记忆 + 打卡 + 心情日记
   - 需要数据库改动，增加用户粘性

3. **第三阶段**（1周）：互动游戏 + 生活助手
   - 需要外部API，长期规划
